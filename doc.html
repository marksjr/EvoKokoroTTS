Evo KokoroTTS

API REST local de Text-to-Speech em portugues brasileiro usando o modelo Kokoro-82M. Converta texto em audio de alta qualidade com multiplas vozes, controle de velocidade e streaming em tempo real.

ⓘ
Base URL: http://localhost:8880
A API tambem gera documentacao interativa automatica em /docs (Swagger UI) e /redoc (ReDoc).

Normalizacao Unicode (NFC)

ⓘ
A API normaliza automaticamente todos os textos para NFC (forma composta Unicode). Isso garante que caracteres como ã, ç, é, ô, ü sejam tratados corretamente, independentemente de como foram digitados ou copiados. Aspas tipograficas, travessoes e reticencias Unicode tambem sao convertidos.

Endpoints

Retorna o estado atual da API, device utilizado e se o modelo esta carregado.

Resposta (200)

{
  "status": "ok",        // "ok" ou "loading"
  "device": "cuda",      // "cuda" ou "cpu"
  "model_loaded": true    // boolean
}

POST /tts Gerar audio completo

Sintetiza o texto completo e retorna o arquivo de audio. Ideal para gerar arquivos de audio para download ou armazenamento.

Request Body (JSON)

CampoTipoObrigatorioPadraoDescricao
text string SIM - Texto para sintetizar. De 1 a 10.000 caracteres.
voice string nao "pf_dora" ID da voz. Opcoes: pf_dora, pm_alex, pm_santa
speed float nao 1.0 Velocidade da fala. Minimo 0.5 (lento), maximo 2.0 (rapido).
format string nao "mp3" Formato de saida: "mp3" (320kbps) ou "wav" (PCM 16-bit)

Payload minimo

{
  "text": "Ola, mundo!"
}

Payload completo

{
  "text": "Ola! Meu nome e Dora e estou testando a API de sintese de voz.",
  "voice": "pf_dora",
  "speed": 1.0,
  "format": "mp3"
}

Resposta (200)

HeaderValor
Content-Typeaudio/mpeg (mp3) ou audio/wav (wav)
Content-Dispositionattachment; filename="tts.mp3"

O corpo da resposta contem os bytes binarios do arquivo de audio.

✓
WAV vs MP3: Use "wav" quando precisar de qualidade maxima sem compressao (editar audio, processar depois). Use "mp3" para distribuicao, web e armazenamento eficiente.
POST /tts/stream Streaming de audio em tempo real

Sintetiza o texto e envia o audio em chunks conforme cada trecho fica pronto. O cliente recebe o primeiro trecho de audio antes de toda a sintese estar concluida. Ideal para playback em tempo real.

Request Body (JSON)

CampoTipoObrigatorioPadraoDescricao
text string SIM - Texto para sintetizar. De 1 a 10.000 caracteres.
voice string nao "pf_dora" ID da voz. Opcoes: pf_dora, pm_alex, pm_santa
speed float nao 1.0 Velocidade da fala. Minimo 0.5, maximo 2.0.

Payload

{
  "text": "Texto longo para streaming em tempo real.",
  "voice": "pm_alex",
  "speed": 1.2
}

Resposta (200 - Streaming)

HeaderValor
Content-Typeaudio/mpeg
Transfer-Encodingchunked
Content-Dispositionattachment; filename="tts_stream.mp3"

O audio e enviado em chunks MP3 a 256kbps conforme cada trecho e sintetizado.

⚠
Streaming: O formato de saida e sempre MP3. Nao ha opcao de formato WAV para streaming. Cada chunk e um segmento MP3 independente.

Exemplos de Uso

cURL

# Verificar status
curl http://localhost:8880/health

# Listar vozes
curl http://localhost:8880/voices

# Gerar MP3 (payload minimo)
curl -X POST http://localhost:8880/tts \
  -H "Content-Type: application/json" \
  -d '{"text":"Ola, mundo!"}' \
  --output audio.mp3

# Gerar MP3 (payload completo)
curl -X POST http://localhost:8880/tts \
  -H "Content-Type: application/json" \
  -d '{"text":"Testando a API Kokoro.","voice":"pm_alex","speed":1.2,"format":"mp3"}' \
  --output audio_alex.mp3

# Gerar WAV
curl -X POST http://localhost:8880/tts \
  -H "Content-Type: application/json" \
  -d '{"text":"Audio em formato WAV.","format":"wav"}' \
  --output audio.wav

# Streaming (salvar)
curl -X POST http://localhost:8880/tts/stream \
  -H "Content-Type: application/json" \
  -d '{"text":"Texto longo para streaming...","voice":"pf_dora","speed":1.0}' \
  --output stream.mp3

# Streaming (playback direto com ffplay)
curl -sN -X POST http://localhost:8880/tts/stream \
  -H "Content-Type: application/json" \
  -d '{"text":"Reproducao em tempo real."}' | ffplay -nodisp -autoexit -

JavaScript / Fetch

// Gerar e reproduzir audio
async function speak(text, voice = "pf_dora", speed = 1.0) {
  const response = await fetch("http://localhost:8880/tts", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ text, voice, speed, format: "mp3" }),
  });

  const blob = await response.blob();
  const url = URL.createObjectURL(blob);
  const audio = new Audio(url);
  audio.play();
}

speak("Ola! Testando a API Kokoro.");


// Streaming com playback em tempo real
async function speakStream(text, voice = "pf_dora", speed = 1.0) {
  const response = await fetch("http://localhost:8880/tts/stream", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ text, voice, speed }),
  });

  // Coletar todos os chunks e reproduzir
  const blob = await response.blob();
  const url = URL.createObjectURL(blob);
  new Audio(url).play();
}


// Download como arquivo
async function download(text, filename = "audio.mp3") {
  const response = await fetch("http://localhost:8880/tts", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ text }),
  });

  const blob = await response.blob();
  const a = document.createElement("a");
  a.href = URL.createObjectURL(blob);
  a.download = filename;
  a.click();
}

Python / Requests

import requests

BASE = "http://localhost:8880"

# Verificar status
health = requests.get(f"{BASE}/health").json()
print(health)
# {'status': 'ok', 'device': 'cuda', 'model_loaded': True}

# Listar vozes
voices = requests.get(f"{BASE}/voices").json()
for v in voices:
    print(f"  {v['id']} - {v['name']} ({v['gender']})")

# Gerar MP3
resp = requests.post(f"{BASE}/tts", json={
    "text": "Ola! Testando a API Kokoro com Python.",
    "voice": "pf_dora",
    "speed": 1.0,
    "format": "mp3",
})
with open("audio.mp3", "wb") as f:
    f.write(resp.content)

# Gerar WAV
resp = requests.post(f"{BASE}/tts", json={
    "text": "Audio em WAV sem compressao.",
    "format": "wav",
})
with open("audio.wav", "wb") as f:
    f.write(resp.content)

# Streaming - salvar chunks conforme chegam
resp = requests.post(f"{BASE}/tts/stream", json={
    "text": "Texto longo para demonstrar o streaming em tempo real.",
    "voice": "pm_alex",
    "speed": 1.1,
}, stream=True)

with open("stream.mp3", "wb") as f:
    for chunk in resp.iter_content(chunk_size=8192):
        f.write(chunk)

# Todas as vozes de uma vez
texto = "Cada voz tem uma personalidade unica."
for voice_id in ["pf_dora", "pm_alex", "pm_santa"]:
    r = requests.post(f"{BASE}/tts", json={"text": texto, "voice": voice_id})
    with open(f"{voice_id}.mp3", "wb") as f:
        f.write(r.content)

Codigos de Erro

CodigoSituacaoDescricao
200 Sucesso Audio gerado com sucesso (binario no corpo da resposta)
400 Bad Request Voz invalida, texto vazio, speed fora do range, formato invalido
422 Validation Error Campos do JSON com tipo errado ou ausentes (gerado pelo FastAPI/Pydantic)
500 Internal Error Erro na sintese do modelo (texto incompativel, modelo nao carregado)

Exemplo de erro 400

{
  "detail": "Voz 'voz_inexistente' nao disponivel. Use GET /voices."
}

Exemplo de erro 422 (validacao)

{
  "detail": [
    {
      "type": "string_too_short",
      "loc": ["body", "text"],
      "msg": "String should have at least 1 character"
    }
  ]
}

Limites e Restricoes

ParametroLimite
Tamanho do texto1 a 10.000 caracteres
Velocidade (speed)0.5 a 2.0
Formatos de saidamp3, wav (apenas /tts)
Formato streamingmp3 apenas (256kbps)
Vozespf_dora, pm_alex, pm_santa
IdiomaPortugues brasileiro (pt-BR) apenas
Requisicoes simultaneas4 (ThreadPoolExecutor)
Chunk maximo interno450 caracteres por chunk de sintese
⚠
Timeout: Textos longos (perto de 10.000 caracteres) podem levar varios segundos para sintetizar, especialmente em CPU. Configure timeout adequado no cliente (recomendado: 120s).
